# Leakage-Aware LoRA Adaptation of SmolVLM for Multimodal Science QA

This repository contains the training, evaluation, and analysis code for the
final project *Leakage-Aware Parameter-Efficient Adaptation of SmolVLM for
Multimodal Science Question Answering* (NYU Tandon, Deep Learning).  It
adapts `HuggingFaceTB/SmolVLM-500M-Instruct` with LoRA (4.34 M trainable
parameters, 0.85% of the base model) using a discriminative choice-ranking
loss, and selects the final checkpoint on a strict group-disjoint validation
split.  Final results: **71.41% strict-validation accuracy**, **79.20%
original-validation accuracy**, **0.76257 public Kaggle score** for the
strict-split single-model submission (best non-probe ensemble: 0.82293).

The full report is at [`report/main.pdf`](report/main.pdf).

## Quick start

```powershell
# 1. Clone and create a Python 3.10+ environment.
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # Windows PowerShell
# source .venv/bin/activate    # Linux / macOS
pip install -r requirements.txt

# 2. Download the SmolVLM backbone (cached for later runs).
python smolvlm_competition.py download

# 3. Build the leakage-aware strict split from data/ (requires raw CSVs).
python smolvlm_competition.py strictsplit `
  --train-csv data/train.csv --val-csv data/val.csv `
  --output-dir data/strict_split_balanced --seed 42

# 4. Train (about 15 minutes on an RTX 5090, ~26 GB peak memory).
.\scripts\repro_main_result.sh        # bash on Linux/macOS
# or run the equivalent CLI command listed in
# submission+artifacts/REPRODUCIBILITY.md.

# 5. Evaluate the saved adapter and (re)generate report tables.
python smolvlm_competition.py eval `
  --checkpoint outputs/r8_choice_split1024_s42_b2/final_adapter `
  --eval-csv data/strict_split_balanced/val.csv
python report/regenerate_metrics.py
```

## Repository layout

| Path | Role |
| --- | --- |
| `smolvlm_competition.py` | Single CLI for data audit, strict split, training, evaluation, prediction, calibration, and ensembling. |
| `submission+artifacts/runs/r8_choice_split1024_s42_b2/` | Final run: configs, logs, metrics, predictions, score details, and submission CSV. |
| `submission+artifacts/legacy/` | Historical Kaggle submissions (probes, ensembles, experiment snapshots). |
| `submission+artifacts/summaries/` | Leaderboard score index and artifact inventory. |
| `submission+artifacts/REPRODUCIBILITY.md` | Step-by-step instructions and exact CLI arguments for every reported run. |
| `report/main.tex` / `report/main.pdf` | Course report (ACL 2023 format). |
| `report/regenerate_metrics.py` | Deterministic metrics regeneration from saved artifacts. |
| `report/error_analysis/` | Self-contained calibration, position-bias, and error-taxonomy scripts. |
| `scripts/repro_main_result.sh` | One-shot end-to-end reproduction script. |

hosted at:

<https://drive.google.com/drive/folders/1g5e3pIBqcSNdKjALeU0k4H7sJ01awRVO?usp=sharing>

and run the evaluation/prediction commands above to reproduce the reported
numbers without retraining.

## Reproducing the report

After running training and evaluation, the report's tables and figures are
regenerated by:

```powershell
python report/regenerate_metrics.py
python report/error_analysis/temperature_scaling.py
python report/error_analysis/position_bias_and_taxonomy.py
```

The first script populates `report/metrics/`, used via
`\input{metrics/report_tables.tex}` in `main.tex`.  The second and third
write JSON outputs that back the calibration and taxonomy claims in the
Error Analysis section.  Build the PDF with:

```powershell
cd report
pdflatex main.tex; bibtex main; pdflatex main.tex; pdflatex main.tex
```
